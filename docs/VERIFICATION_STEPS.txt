Verification steps for subscriptions and general checks

Prerequisites:
- Ensure Redis is running and reachable at the configured `REDIS_HOST`/`REDIS_PORT`.
- Ensure the gateway and all subgraphs are running:
  - Gateway: `http://localhost:4000/graphql` (WS: `ws://localhost:4000/graphql`)
  - User: `http://localhost:4001/graphql`
  - Post: `http://localhost:4002/graphql`
  - Feed: `http://localhost:4003/graphql`

Database Setup:
- Before running services, ensure your PostgreSQL database is initialized:
  ```bash
  psql -h localhost -U your_user -d your_db -f sql-scripts/setup.sql
  ```
- Alternatively, the services will fall back to an in-memory SQLite database if environment variables are not set.

Simple checks:
1. HTTP introspection (gateway):
   - `curl -X POST -H "Content-Type: application/json" --data '{"query":"{ __schema { types { name } } }"}' http://localhost:4000/graphql`

2. Verify `_service.sdl` per subgraph (to ensure federation fields are exposed):
   - `curl -s -X POST -H 'Content-Type: application/json' --data '{"query":"{ _service { sdl } }"}' http://localhost:4001/graphql | jq .`

Subscription verification (Node.js client using `graphql-ws`):

1. Install the client (from repo root or a temp folder):
   ```bash
   npm init -y
   npm install graphql-ws ws
   ```

2. Create `subscribe.js` with the following content:
   ```js
   const { createClient } = require('graphql-ws');
   const WebSocket = require('ws');

   const client = createClient({
     url: 'ws://localhost:4000/graphql',
     webSocketImpl: WebSocket,
   });

   (async () => {
     const onNext = (data) => console.log('Received', JSON.stringify(data, null, 2));

     console.log('Starting subscription...');
     const dispose = client.subscribe(
       {
         query: `subscription OnPostCreated { postCreated { id title content userId } }`,
       },
       {
         next: onNext,
         error: (err) => console.error('Err', err),
         complete: () => console.log('Completed'),
       }
     );

     // To test User registration:
     // query: `subscription OnUserRegistered { userRegistered { id name email } }`
     
     // To test Feed creation:
     // query: `subscription OnFeedCreated { feedCreated { id title body userId } }`
   })();
   ```

3. Run the subscriber:
   ```bash
   node subscribe.js
   ```

4. In another terminal, trigger a mutation via the Gateway (port 4000):
   ```bash
   # Create a post
   curl -X POST -H "Content-Type: application/json" --data '{"query":"mutation { createPost(title:\"hello\", content:\"world\", userId:1){ id title } }"}' http://localhost:4000/graphql
   ```

If the subscription prints the new post, end-to-end subscription is working through the gateway.

Troubleshooting tips:
- ERR_PACKAGE_PATH_NOT_EXPORTED: If you encounter this error, ensure your import is `require('graphql-ws/use/ws')` or check the latest `graphql-ws` documentation for ESM/CJS export changes.
- Port Conflict: Use `lsof -i :4000` to check if the port is already in use.
- Redis Connection: Ensure all services show "Redis connected" in their logs.
